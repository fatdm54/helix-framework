# 模板集合

> 常用模板，快速复用

---

## 1. 任务模板 (task-template.md)

```markdown
## Task: {任务名称}

### 需求
{详细描述}

### 验收标准
- [ ] 标准 1
- [ ] 标准 2

### 约束
- Context 限制: {N} tokens
- 预计耗时: {M} 分钟
- 优先级: {P0/P1/P2}

### 日志
- 开始时间: {时间}
- 状态: {pending|in_progress|done|blocked}

### 反思（完成后填写）
- 做得好:
- 改进点:
- 下次行动:
```

## 2. 调研模板 (research-template.md)

```markdown
# 调研: {主题}

## 问题
{要回答的核心问题}

## 调研方法
- [ ] Web 搜索
- [ ] 文档阅读
- [ ] 对比分析

## 关键发现

### 发现 1
- 内容: {描述}
- 来源: [URL](link)
- 可信度: {高/中/低}

### 发现 2
- 内容: {描述}
- 来源: [URL](link)
- 可信度: {高/中/低}

## 结论
{总结 + 建议}

## 待验证
- [ ] 待验证点 1
- [ ] 待验证点 2

## 归档
- 日期: {日期}
- 有效期: {3个月/6个月/1年}
```

## 3. 代码审查模板 (review-template.md)

```markdown
# Code Review: {PR/Commit 名称}

## 基本信息
- 作者: {author}
- 文件: {files}
- 行数: +{additions} -{deletions}

## 审查结果

### 必须修复 (Blocker)
| 位置 | 问题 | 建议 |
|------|------|------|
| line 45 | 安全漏洞 | 使用参数化查询 |

### 建议修复 (Major)
| 位置 | 问题 | 建议 |
|------|------|------|
| line 78 | 性能问题 | 添加缓存 |

### 可选优化 (Minor)
- 代码风格建议
- 注释建议

## 总体评价
- [ ] 可以合并
- [ ] 需要修改
- [ ] 拒绝合并

## 备注
{其他说明}
```

## 4. 测试报告模板 (test-template.md)

```markdown

## 测试概况
- 总测试数: {N}
- 通过: {M}
- 失败: {K}
- 覆盖率: {X}%

## 通过的测试
- [ ] test_xxx
- [ ] test_yyy

## 失败的测试
| 测试名 | 原因 | Issue |
|--------|------|-------|
| test_zzz | 预期行为未实现 | BUG-001 |

## 风险评估
- 高风险: {原因}
- 中风险: {原因}

## 建议
{改进建议}
```
